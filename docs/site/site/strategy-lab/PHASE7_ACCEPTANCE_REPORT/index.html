    4. Kimi-2.5 Integration

    4.1 Connectivity

    Endpoint: Internal Ollama bridge
    Format: OpenAI Completions API
    Test Result: 

    HTTP 200 OK
Response Keys: ['id', 'object', 'created', 'model', 'system_fingerprint', 'choices', 'usage']
Model: kimi-k2.5:cloud
    Status: ✅ REACHABLE

    4.2 Parser Robustness

    Multi-Format Support: 1. OpenAI Format: choices[0].message.content 2. Ollama Direct: message.content 3. Alternative: response

    JSON Extraction Strategy: 1. Code-Block mit json tag: ```json {...} ``` 2. Code-Block ohne tag: ```{...} ``` 3. Raw JSON: first { to last } 4. Whole response as JSON

    Error Handling: - API errors → {"error": "HTTP 401: ..."} - Parse errors → {"parse_error": true, "raw_preview": "..."} - No silent failures

    4.3 Fallback Analysis

    Trigger: API unavailable / no key configured
    Behavior: Heuristic evaluation based on scorecard metrics
    Output: Valid AnalystReport with confidence 0.5, verdict TWEAK/FAIL/PASS based on thresholds

    Thresholds: - PASS: trade_count ≥30, return >5%, drawdown >-20%, PF>1.3, WR>45%, exp>0 - FAIL: trade_count <10 or return <-5% - TWEAK: everything else

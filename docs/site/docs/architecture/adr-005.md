# ADR-005: Three-Layer Architecture + Sentiment Risk Layer

**Status:** APPROVED  
**Updated:** 2026-04-20

## Decision

The system uses a Three-Layer Architecture with a Sentiment Risk Layer for V2+.

## Layer 1: Foundry (Backtesting)

MACD Momentum + ADX+EMA Regime Filter — **FROZEN** during paper trading.

- Entry: `macd_hist > 0 AND close > ema_50 AND ema_50 > ema_200 AND adx_14 > 20`
- Exit: Trailing Stop 2%, SL 2.5%, Max Hold 48 Bars
- Gate: CL≤12, DD≤20%, PF≥1.05, Return≥1%, Sharpe≥0.1, Trades≥20
- Validation: 75% Pass Rate (12/16), avg DD 14.1%, avg CL 6.5

## Layer 2: Executor (Live Trading)

Paper Engine V1 — running on Hyperliquid Testnet since 2026-04-20.

- 7 Modules: data_feed, signal_generator, state_manager, risk_guard, paper_engine, discord_reporter, command_listener
- 83 Tests (75 Unit + 7 E2E + 1 Regression), 100% Pass
- 5 Bug Audits completed (17 bugs found & fixed)
- Guard States: RUNNING → SOFT_PAUSE → STOP_NEW → KILL_SWITCH → COOLDOWN → RUNNING
- Kill-Switches: CL≥5, DD>20%, DailyLoss>5%
- Leverage Tiers (ADR-007): BTC/ETH 1.8x, SOL/DOGE/ADA 1.5x, AVAX 1.0x

## Layer 3: Monitor (Planned)

Dashboard, equity curve per bar, daily reports, alerting.

## Sentiment/Macro Risk Layer (V2+)

**KI als Signalgeber = YES**, KI als Richter = NO

| Aspect | Decision |
|--------|----------|
| Role | Signal Filter, nicht Trade-Entscheidung |
| Score | 0-100 (0=bearish, 100=bullish) |
| Format | JSON-Mode für deterministische Extraktion |
| Fail-safe | Bei Fehler → ignorieren (nicht handeln ist sicherer) |
| Direction | Nur Downsizing, nie Upsizing |
| Priority | V2 High für 1h Timeframe |

### V2 Sentiment Sources (Priority)

1. **Crypto-specific news** (CoinDesk, CryptoSlate) — High Priority
2. **Macro indicators** (Fed, CPI, DXY) — Medium Priority
3. **Social sentiment** (Fear&Greed Index, Reddit) — Low Priority
4. **On-chain data** (Exchange Netflow 7-14d) — Medium Priority

### V2 Alpha Stack (Priority)

1. Asset-Ranking (ROC-basiert) — Hoch
2. ADX-based Position Sizing — Hoch
3. 2x Leverage für Blue Chips — Mittel
4. Limit-Orders statt Market — Mittel
5. HTF-Alignment — Niedrig
6. ~~ATR-Stops~~ — **ABGELEHNT**
7. ~~Kelly Criterion~~ — **ABGELEHNT**

## Explicitly Rejected

- ATR-based stops (no improvement over baseline)
- Kelly criterion (unstable win-rate in crypto)
- 3x leverage (DD 29-51%, constant KILL triggers)
- Dynamic auto-derate (too error-prone for V1)
- KI als Richter (AI decides trade direction)

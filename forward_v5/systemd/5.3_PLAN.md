## Scope

Web-basiertes Dashboard für Echtzeit-Monitoring des Forward Services.

### 5.3.1: Health Status Endpoint
- [ ] JSON endpoint `/health` für externe Monitoring-Systeme
- [ ] Memory, CPU, Uptime, Event Store Status
- [ ] Circuit Breaker State
- [ ] Last Heartbeat timestamp

### 5.3.2: Simple Web Dashboard
- [ ] Static HTML dashboard
- [ ] Auto-refresh (10s interval)
- [ ] Memory graph (last 60min)
- [ ] Event count, error rate
- [ ] Mobile responsive

### 5.3.3: Alert Integration
- [ ] Webhook support for CRITICAL events
- [ ] Optional: Discord/Slack notifications
- [ ] Alert history log

## Architecture

```
┌─────────────────────────────────────┐
│  Browser / Mobile                   │
│  (Dashboard UI)                     │
└──────────────┬──────────────────────┘
               │ HTTP GET /health
               ▼
┌─────────────────────────────────────┐
│  Node.js Express Server (optional)  │
│  - Serves static dashboard          │
│  - /health endpoint                 │
│  - Webhook dispatch                 │
└──────────────┬──────────────────────┘
               │ Reads
               ▼
┌─────────────────────────────────────┐
│  runtime_validation/state.json      │
│  runtime/event_store.db             │
└─────────────────────────────────────┘
```

---

## Technical Decisions

- **Framework:** Express.js (already in Node ecosystem)
- **Port:** 3000 (configurable via PORT env)
- **Auth:** None for v1 (localhost/VPN only)
- **Update Interval:** 10 seconds (WebSocket optional future)
- **Data Source:** state.json + event_store.db
